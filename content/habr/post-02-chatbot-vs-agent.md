# Почему твой чат-бот — игрушка, а не работник. Гайд по автономным агентам

В 2024 все делали чат-ботов. В 2025 — копайлотов. В 2026 наступает эра автономных агентов. И большинство до сих пор путает эти три понятия.

Давайте раз и навсегда разложим, чем отличается чат-бот от копайлота от автономного агента. И почему «агент» — это не просто модное слово, а принципиально другая архитектура.

## Три уровня автономности

### Уровень 0: Чат-бот

```
Пользователь → [сообщение] → Бот → [ответ] → Пользователь
```

Бот ждёт. Ты написал — он ответил. Ты молчишь — бот мёртв.

**Характеристики:**
- Работает строго в формате «запрос → ответ»
- Не имеет памяти между сессиями
- Не может инициировать действие сам
- Не имеет доступа к внешним инструментам (или имеет минимальный)

**Пример:** ChatGPT в базовом режиме. Ты спрашиваешь — он отвечает. Закрыл вкладку — диалог окончен.

### Уровень 1: Копайлот

```
Пользователь → [контекст + запрос] → Копайлот → [предложение / черновик] → Пользователь → [решение]
```

Копайлот подсказывает, но не действует. Он видит контекст (твой код, твой документ) и предлагает варианты. Решение всегда за тобой.

**Характеристики:**
- Видит контекст (файлы, код, экран)
- Предлагает, но не исполняет без подтверждения
- Работает только когда пользователь активен
- Всегда есть человек в цикле принятия решений

**Пример:** GitHub Copilot. Видит твой код, предлагает автодополнение. Но не коммитит и не деплоит без тебя.

### Уровень 2: Автономный агент

```
Событие → Агент → [анализ] → [решение] → [действие] → [результат]
                ↑
        [память + навыки + инструменты]
```

Агент живёт своей жизнью. Сработал триггер (новое письмо, время по расписанию, вебхук) — агент просыпается, анализирует ситуацию, принимает решение и действует. Без человека в цикле.

**Характеристики:**
- Работает по триггерам (события, расписание, вебхуки)
- Имеет постоянную память (SQLite, векторная база)
- Имеет набор инструментов (API, файловая система, браузер)
- Принимает решения самостоятельно в заданных границах
- Может делегировать суб-агентам

**Пример:** Агент-рекрутер на OpenClaw. Каждый час проверяет HH по фильтрам → находит новых кандидатов → проверяет резюме → отправляет отклик → записывает в таблицу → шлёт отчёт в Telegram. Без участия человека.

## Почему это важно для разработчика

**Чат-ботов** уже миллион. Рынок перенасыщен. Сделать бота может любой студент за вечер. Цена → 0.

**Копайлотов** делают крупные компании (GitHub, Cursor). Вход в нишу требует интеграции в IDE/платформу.

**Автономных агентов** — единицы. Спрос растёт быстрее предложения. Компании готовы платить за автоматизацию процессов, которые раньше делали люди. Агент, который заменяет ассистента за 50 000 ₽/мес, окупается за неделю. Цена агента: 15 000–150 000 ₽.

## С чего начать

1. **Понять процесс.** Найди в своей работе рутину, которая повторяется каждый день. Это кандидат на автоматизацию.
2. **Выбрать инструмент.** OpenClaw (Python, open-source) + Ollama (локальные LLM) = 0 ₽ на старте.
3. **Собрать MVP.** Минимального агента можно собрать за вечер. Если он экономит час в день — он уже окупился.
4. **Показать клиенту.** Не продавай код. Продавай результат: «твой ассистент тратит 3 часа в день на рассылку. Агент делает это за 5 минут».
