# Агент-рекрутер на OpenClaw: сортирует резюме, пишет кандидатам, заполняет таблицу

Каждый HR тратит 3-4 часа в день на рутину: открыть HH, проверить новых кандидатов, сравнить с фильтрами, отправить отклик, записать в таблицу. Ровно эту работу я автоматизировал за вечер. Агент делает всё сам: мониторит площадку, фильтрует кандидатов, отправляет сообщения и ведёт таблицу.

## Что получится

Агент-рекрутер, который:
- Каждый час проверяет HH по заданным фильтрам (должность, зарплата, город, ключевые слова)
- Сравнивает с уже просмотренными (SQLite-память)
- Для новых кандидатов: анализирует резюме через LLM и выставляет оценку fit
- Отправляет уведомление в Telegram с краткой сводкой
- Записывает результат в Google Таблицу

## Инструменты

| Инструмент | Для чего | Стоимость |
|-----------|----------|-----------|
| Python 3.11+ | Язык разработки | 0 ₽ |
| OpenClaw | Фреймворк агента | 0 ₽ |
| Ollama + Qwen 2.5 | Локальная LLM для анализа резюме | 0 ₽ |
| SQLite | Память агента (просмотренные кандидаты) | 0 ₽ |
| Telegram Bot API | Интерфейс и уведомления | 0 ₽ |
| HH API | Доступ к вакансиям/резюме | 0 ₽ |

## Шаг 1: Установка и настройка OpenClaw

```bash
# Клонируем фреймворк
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# Устанавливаем зависимости
pip install -r requirements.txt

# Проверяем: OpenClaw должен быть доступен
openclaw --version
```

## Шаг 2: Установка Ollama и загрузка модели

```bash
# Установка Ollama
curl -fsSL https://ollama.com/install.sh | sh  # Linux/Mac
# или скачать установщик с ollama.com для Windows

# Загружаем Qwen 2.5 (отличный баланс качество/скорость для русского языка)
ollama pull qwen2.5:7b

# Проверяем
ollama list
```

## Шаг 3: Создаём workspace агента

```bash
mkdir -p ~/.openclaw/workspace/recruiter/skills
cd ~/.openclaw/workspace/recruiter
```

Файл `AGENTS.md`:

```markdown
# Агент-рекрутер

Ты — рекрутер. Твоя задача: мониторить вакансии на hh.ru, анализировать резюме
кандидатов и находить подходящих по заданным критериям.

Критерии отбора хранятся в skills/search/SKILL.md.
```

Файл `SOUL.md`:

```markdown
# Личность

Ты — методичный и внимательный рекрутер. Ты не пропускаешь ни одной детали
в резюме, но не перегружаешь заказчика лишней информацией. Твой девиз:
"Лучше пять точных кандидатов, чем пятьдесят случайных".
```

## Шаг 4: Пишем навык поиска

Создаём `skills/search/SKILL.md`:

```yaml
---
name: hh-search
description: Поиск вакансий/резюме на hh.ru с фильтрацией и сохранением в историю
---
```

Текст навыка (body):

```markdown
# Поиск кандидатов на HH

## Алгоритм

1. Проверить расписание. Если прошло менее 1 часа с последнего поиска — выйти.
2. Запросить HH API. Использовать сохранённые фильтры из filters.json.
3. Сверить с историей. Для каждой найденной вакансии/резюме проверить
   history.db — не обрабатывали ли мы её раньше.
4. Новые — проанализировать. Отправить текст резюме в LLM и получить оценку
   по критериям:
   - Соответствие должности (0-10)
   - Релевантный опыт (лет)
   - Ключевые навыки из фильтра (список)
5. Результат. Новые подходящие кандидаты → уведомление в Telegram.
   Все обработанные → запись в history.db.

## Фильтры (filters.json)

{
  "position": "Python Developer",
  "salary_min": 150000,
  "city": "Москва",
  "keywords": ["Python", "Django", "FastAPI", "PostgreSQL", "Docker"],
  "experience_min_years": 2
}
```

## Шаг 5: Пишем навык уведомлений

Создаём `skills/notify/SKILL.md`:

```yaml
---
name: telegram-notify
description: Отправка уведомлений о новых кандидатах в Telegram
---
```

Тело навыка:

```markdown
# Уведомление в Telegram

Формат сообщения о новом кандидате:

🔔 **Новый кандидат: {должность}**

👤 Имя: {имя}
💼 Опыт: {N} лет
🎯 Fit: {X}/10
📍 Город: {город}
💸 Зарплата: {сумма} ₽

🔑 Ключевые навыки: {навыки}

📎 Ссылка на резюме: {url}
```

## Шаг 6: Конфигурация openclaw.json

```json
{
  "gateway": {
    "port": 8080
  },
  "agents": {
    "defaults": {
      "model": "ollama/qwen2.5:7b",
      "workspace": "~/.openclaw/workspace/recruiter"
    },
    "list": [
      {
        "id": "recruiter",
        "name": "Recruiter Agent",
        "skills": ["hh-search", "telegram-notify"],
        "triggers": [
          {
            "type": "cron",
            "schedule": "0 * * * *"
          }
        ]
      }
    ]
  }
}
```

## Шаг 7: Запуск

```bash
# Запускаем gateway
openclaw gateway start

# Проверяем статус
openclaw gateway status

# Агент recruiter запущен и будет проверять HH каждый час
```

## Результат

Каждый час в Telegram приходит сообщение:

> 🔔 Новый кандидат: Python Developer
> 👤 Имя: Дмитрий С.
> 💼 Опыт: 4 года
> 🎯 Fit: 8/10
> 📎 Ссылка на резюме

HR открывает сообщение, видит топ-кандидатов и точечно с ними работает. Вместо 3 часов мониторинга — 5 минут на осмысленные действия.
